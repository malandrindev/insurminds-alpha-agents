# Regras de colaboração

## Responsabilidade por branch

### Vilela
Branch: `feature/desafio3-eda-modelagem-vilela`

Escopo:
- análise exploratória;
- gráficos;
- tratamento de valores ausentes;
- engenharia de atributos;
- Regressão Logística;
- interpretação técnica inicial;
- comparação dos modelos.

### Vitor
Branch: `feature/desafio3-pipeline-integracao-vitor`

Escopo:
- estrutura do notebook;
- validações de entrada;
- pipeline de pré-processamento;
- Random Forest;
- integração;
- geração do `submission.csv`;
- submissão ao Kaggle;
- pacote final.

### Wagner
Branch: `docs/desafio3-relatorio-wagner`

Escopo:
- checklist dos requisitos;
- introdução;
- metodologia em linguagem de negócio;
- interpretação dos gráficos;
- resultados;
- conclusão;
- limitações;
- validação funcional.

## Pull Requests

Todo Pull Request deve informar:

1. O que foi alterado.
2. Como testar.
3. Resultado esperado.
4. Pendências conhecidas.
5. Evidências, quando aplicável.

## Regra de revisão

- Código do Vitor: revisão do Vilela.
- Código do Vilela: revisão do Vitor.
- Documentação do Wagner: revisão de um integrante técnico.
- Entrega final: aprovação dos três integrantes.
